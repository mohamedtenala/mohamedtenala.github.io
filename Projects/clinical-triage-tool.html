PROCESS AUTOMATION

Clinical Triage & Doctor Directory

An internal web application developed to help call center and front-desk teams recommend the appropriate medical specialty and quickly identify suitable physicians.

INDUSTRY Healthcare
PLATFORM Python · Streamlit
AUDIENCE Call Center & Front Desk
FOCUS Triage · Search · Process Improvement

BUSINESS CHALLENGE

Patient routing depended heavily on individual staff knowledge.

Call center and front-desk staff needed to identify the correct medical specialty based on patient symptoms while also searching for physicians with relevant expertise. The process was manual, inconsistent and time-consuming.

MY SOLUTION

Built a rule-based triage and physician search application.

I designed and developed an internal application that combines symptom-to-specialty decision logic with a searchable doctor directory and an administration interface.

  • Created symptom-based specialty recommendation rules.
  • Added follow-up questions for specific clinical scenarios.
  • Included trauma and urgency decision logic.
  • Displayed available physicians for the recommended specialty.
  • Built a searchable doctor directory.
  • Included physician specialty, expertise and profile information.
  • Created an administration module for content maintenance.
  • Stored structured application data using SQLite.

APPLICATION MODULES

Three integrated modules supporting one workflow.

01 Symptom-to-specialty triage engine
02 Searchable doctor directory
03 Administration and content management
Real-Time Specialty and physician recommendations

MY ROLE

Owned the project from business problem to deployment.

I gathered operational requirements, designed the business logic, created the application structure, developed the tool using Python and Streamlit, configured the database and prepared the solution for internal deployment.

  • Business requirements gathering
  • Workflow and decision-rule design
  • User interface development
  • Database design and content structure
  • Testing and validation
  • Internal deployment preparation

BUSINESS VALUE

Standardized routing and reduced time spent searching.

The application provides staff with a consistent method for identifying the appropriate specialty while reducing reliance on memory, spreadsheets and manual physician searches.

  • Supports more consistent specialty recommendations.
  • Reduces time spent searching for physician information.
  • Improves first-contact routing.
  • Provides one centralized source of doctor information.
  • Simplifies onboarding for new call center and front-desk staff.
